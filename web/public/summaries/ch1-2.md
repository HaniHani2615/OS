## CHƯƠNG 1 — GIỚI THIỆU HỆ ĐIỀU HÀNH

### 1.1 Hệ điều hành là gì?

- **HĐH** = chương trình **trung gian** giữa người dùng và phần cứng.
- 2 chức năng cốt lõi:
  1. **Quản lý & cấp phát hiệu quả tài nguyên phần cứng** (CPU, bộ nhớ, I/O…)
  2. **Thực thi chương trình người dùng**, cung cấp môi trường tiện lợi.

### 1.2 Kiến trúc máy tính

- **Kiến trúc Von Neumann**: CPU + Bộ nhớ + I/O qua bus chung.
- Phân cấp lưu trữ (nhanh→chậm, nhỏ→lớn): **Thanh ghi → Cache → RAM → Đĩa cứng → Băng từ**.

**Phân loại hệ máy tính:**

| Loại | Đặc điểm |
|---|---|
| Đơn vi xử lý (single-processor) | 1 CPU |
| Đa vi xử lý (multiprocessor) = song song = đa lõi | nhiều CPU/lõi → tăng tốc, tiết kiệm chi phí, độ tin cậy ↑ |
| Đối xứng (SMP) | tất cả CPU ngang hàng |
| Không đối xứng | có CPU master + slave |

### 1.3 Lịch sử (4 thế hệ)

1. **1945–1955**: ống chân không (vacuum tube), lập trình bằng dây
2. **1955–1965**: transistor, xử lý theo lô
3. **1965–1980**: IC, đa chương trình, time-sharing
4. **1980–nay**: VLSI, PC, mạng, di động, đám mây

### 1.4 Các thuật ngữ quan trọng

| Thuật ngữ | Nghĩa |
|---|---|
| **Batch processing** (xử lý theo lô) | gom job → chạy lần lượt, không tương tác |
| **Multiprogramming** (đa chương trình) | nhiều chương trình trong job pool. Khi CT đang chạy bị I/O / kết thúc / ngắt → CT khác chiếm CPU |
| **Time-sharing / multitasking** (phân chia thời gian) | mở rộng đa CT — CPU luân phiên giữa nhiều CT, mỗi CT 1 lát thời gian (time slice) |
| **Parallel** (xử lý song song) | nhiều CPU thực thi đồng thời |
| **Real-time** (thời gian thực) | có ràng buộc về thời gian phản hồi |
| **Embedded** (nhúng) | trong thiết bị chuyên dụng |
| **Special-purpose** (chuyên dụng) | thiết kế cho một mục đích cụ thể |

### 1.5 Một số HĐH

- **UNIX** (1970), Solaris, **Linux**
- **Windows** NT/7/8/10 (từ 1989)
- **macOS / Mac OS X** (từ 1984)
- **Mobile**: iOS, Android

### 1.6 Kiến thức mở rộng

- Hệ phân cụm (clustered), Hệ phân tán (distributed), Cloud computing, Mobile computing
- Quantum computing: $n$ qubit biểu diễn $2^n$ giá trị đồng thời

## CHƯƠNG 2 — CẤU TRÚC HỆ ĐIỀU HÀNH

### 2.1 Các dịch vụ của HĐH

**Dịch vụ cho người dùng:**
- Cung cấp giao diện
- Thực thi chương trình
- Thao tác vào/ra
- Quản lý vào/ra
- Truyền thông
- Phát hiện lỗi

**Dịch vụ cho hệ thống:**
- Cấp phát tài nguyên
- Thống kê
- Bảo vệ & an ninh

### 2.2 Giao diện người dùng ↔ HĐH

- **CLI (Command Interpreter)**: dòng lệnh
- **GUI (Graphical UI)**:
  - **Aero** — Windows Vista/7/8/10
  - **Aqua** — macOS X
  - **CDE** — Unix/Linux/OpenVMS
  - **KDE** — Unix
  - **GNOME** — Ubuntu

### 2.3 Hàm hệ thống (System Call) & API

- **System call** = giao diện lập trình → các dịch vụ HĐH cung cấp.
- Viết bằng **C / C++ / hợp ngữ**.
- CT người dùng truy cập gián tiếp qua **API** (Application Program Interface).
  - Ví dụ: `read()` trong UNIX là API gọi system call tương ứng.

### 2.4 Chương trình hệ thống (System programs)

Cung cấp môi trường thuận tiện để phát triển/thực thi chương trình:
- **Quản lý tệp tin**: tạo/xoá/copy/đổi tên/in
- **Thông tin trạng thái**: thời gian, RAM, số người dùng…
- **Hỗ trợ ngôn ngữ**: compiler, interpreter, debugger
- **Sửa đổi tệp**: editor
- **Nạp & thực thi chương trình**
- **Truyền thông**: kết nối ảo
- **Dịch vụ phía sau** (background)

### 2.5 ⭐ Cấu trúc HĐH (5 kiểu — RẤT HAY THI)

| Cấu trúc | Mô tả | Ví dụ |
|---|---|---|
| **Đơn giản** (Simple) | gộp tất cả vào ít module, ít phân tách quyền | **MS-DOS** |
| **Nguyên khối** (Monolithic) | nhân lớn, khó bảo trì | UNIX phiên bản đầu |
| **Phân tầng** (Layered) | chia thành nhiều layer, layer trên gọi layer dưới | – |
| **Vi nhân** (Microkernel) | nhân chỉ giữ chức năng tối thiểu, phần khác qua **trao đổi thông điệp** | **Mach, Windows NT, QNX** |
| **Module** ⭐ tốt nhất hiện nay | nạp module động vào nhân | **Solaris, Linux, macOS X, Windows** |

### 2.6 Máy ảo (Virtual Machine)

- HĐH thường ≠ Máy ảo: máy ảo cung cấp giao diện y hệt phần cứng → có thể chạy nhiều HĐH khách trên cùng phần cứng vật lý.
- Ví dụ: **VMware**, **JVM** (Java Virtual Machine).

## ⚠️ LƯU Ý ÔN TẬP

1. **Phân biệt** đa chương trình (multiprogramming) ≠ phân chia thời gian (time-sharing): đa CT chỉ chuyển khi CT đang chạy bị **block**; time-sharing chuyển theo **đồng hồ** (quantum).
2. **Microkernel ≠ Module**: microkernel **giảm** kích thước nhân và đẩy ra user space; module **giữ** kích thước nhân nhưng nạp **động**.
3. **System call vs API**: API = lớp che giấu, system call = giao diện thật. Lập trình viên thường gọi API.
4. **MS-DOS = Simple structure**, **UNIX đời đầu = Monolithic**, **Linux/Windows hiện đại = Module**.
5. Ôn quy đổi: **bit, byte, KB, MB, GB, TB, PB, EB, ZB, YB**; ký hiệu **Mbps vs MB/s** (1 byte = 8 bit).
6. Hệ cơ số: **2, 4, 8, 16 (hexa)** — câu hỏi tính địa chỉ dùng nhiều ở chương 7–8.
